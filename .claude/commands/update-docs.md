# update-docs スキル

変更内容に基づいて詳細設計書を更新し、コミット・プッシュまで一連で実行します。

## 実行手順

以下の手順を順番に実行してください。

### ステップ1: 変更内容の確認

`git diff HEAD` と `git status` を実行して、今回の変更対象ファイルと変更内容を把握してください。

### ステップ2: 更新対象章の特定

下記の対応表をもとに、更新が必要な詳細設計書の章を特定してください。

| 変更したファイル・内容 | 更新対象章 |
|---|---|
| `api/db/schema.ts` | 4_データベース設計.md |
| `api/server.ts`（エンドポイント追加・変更） | 5_API設計.md |
| 認証・セッション・ロック処理 | 6_認証・セキュリティ設計.md |
| `src/components/**/*.vue`（画面・フォーム変更） | 3_画面設計.md |
| エラーハンドリング・楽観的ロック・ページネーション・ソート・フィルタ・Excel仕様 | 7_共通仕様.md |
| `package.json`・デプロイ設定・環境変数 | 8_開発・運用.md |
| アーキテクチャ・技術スタック変更 | 2_システム概要.md |

### ステップ3: 詳細設計書の更新

特定した章（`docs/詳細設計書/` 配下のMarkdownファイル）を読み込み、変更内容を反映して更新してください。

更新ルール：
- 追加した機能・フィールドは設計書に追記する
- 削除・変更した仕様は設計書から削除・修正する
- テーブル・フロー図・コードサンプルも最新の実装に合わせて修正する

### ステップ4: コミット・プッシュ

以下の形式でコミットメッセージを作成し、コードと設計書をまとめてコミット・プッシュしてください。

```
<type>: <変更内容の要約（日本語可）>

docs: 詳細設計書 <章番号> を更新
```

typeの選択基準：
- `feat`: 新機能追加
- `fix`: バグ修正
- `refactor`: リファクタリング
- `style`: スタイル変更
- `docs`: 設計書のみの変更

ブランチ：`claude/pensive-pasteur-DkcpT` にプッシュしてください。

### ステップ5: 完了報告

以下の形式で完了を報告してください。

```
✅ 変更・デプロイ・設計書更新が完了しました。

【変更内容】
- （変更した内容を箇条書き）

【更新した設計書】
- docs/詳細設計書/X_章名.md

【コミット】
- <コミットハッシュ>
```

# 3. 画面設計

## 3.1 画面一覧

| 画面名 | URL | コンポーネント | アクセス権 | 説明 |
|---|---|---|---|---|
| ログイン画面 | `/login` | `LoginView.vue` | 全ユーザー（認証不要） | メール・パスワードでログイン |
| ホーム（一覧） | `/` | `HomeView.vue` | ログイン済みユーザー | レコード一覧・検索・フィルタ |
| ユーザー管理 | `/users` | `UsersView.vue` | 管理者（admin）のみ | ユーザーの追加・編集・削除 |

## 3.2 画面遷移図

```
未認証
  └─→ /login（ログイン画面）
          │ ログイン成功
          ▼
        /（ホーム画面）
          │
          ├─ [新規登録] ボタン → レコード登録モーダル（CustomerModal）
          │                         ├── 基本情報タブ
          │                         ├── 案件情報タブ
          │                         ├── 営業情報タブ
          │                         ├── TODOタブ
          │                         └── 月次スケジュールタブ
          │
          ├─ [編集] ボタン → レコード編集モーダル（同上）
          ├─ [閲覧] ボタン → レコード閲覧モーダル（読み取り専用）
          │
          └─ [ユーザー管理] リンク（管理者のみ表示）
                │
                ▼
              /users（ユーザー管理画面）
                └─ [← 一覧に戻る] → /（ホーム画面）
```

**ナビゲーションガード仕様：**
- 未ログイン状態で `/` または `/users` にアクセス → `/login` にリダイレクト
- 一般ユーザー（role=user）が `/users` にアクセス → `/` にリダイレクト

## 3.3 ログイン画面（`/login`）

### 入力項目

| フィールド名 | 型 | 必須 | 属性 | 説明 |
|---|---|---|---|---|
| メールアドレス | email | ○ | autocomplete="email" | @sas-com.com ドメインのアカウント |
| パスワード | password | ○ | autocomplete="current-password" | 表示/非表示の切替ボタンあり |

### 動作仕様

- ログインボタン押下 → `POST /api/auth/login` を呼び出し
- 成功時：セッションCookieが発行され、`/` へリダイレクト
- 失敗時：エラーメッセージをフォーム下部に表示

### エラーメッセージ一覧

| 状況 | 表示メッセージ |
|---|---|
| 認証失敗 | 「メールアドレスまたはパスワードが正しくありません。」 |
| アカウント無効 | 「このアカウントは無効です。管理者にお問い合わせください。」 |
| アカウントロック中 | 「アカウントがロックされています。N分後に再試行してください。」 |

## 3.4 ホーム画面（`/`）

### 画面構成

1. **共通ヘッダー（`AppHeader.vue`）**
   - システム名
   - Excelエクスポートボタン
   - Excelインポートボタン
   - ログアウトボタン
   - ユーザー管理リンク（管理者のみ表示）

2. **フィルタ・検索エリア**
3. **レコード一覧テーブル（`CustomerTable.vue`）**
4. **ページネーション**
5. **新規登録ボタン**

### フィルタ・検索仕様

| 項目 | 種別 | 検索方式 |
|---|---|---|
| キーワード検索 | テキスト入力 | 顧客名・案件名の部分一致（大文字小文字区別なし） |
| 担当部署 | ドロップダウン | 完全一致（DB上の既存値から動的生成） |
| 商流 | ドロップダウン | 完全一致 |
| 契約形態 | ドロップダウン | 完全一致 |
| 客先グレード | ドロップダウン | 完全一致 |

### 一覧テーブル表示カラム

| カラム | フィールド | ソート | 表示形式 |
|---|---|---|---|
| No | `no` | ○ | 数値 |
| 顧客名 | `customerName` | ○ | テキスト |
| 売上高（百万円） | `revenue` | ○ | 整数（3桁区切り） |
| IT比率（%） | `itPartnerRatio` | ○ | 小数（`toFixed(1)`） |
| SAS比率（%） | `sasRatio` | ○ | 小数 |
| 参画人数 | `memberCount` | ○ | 整数 |
| 平均単価（円） | `avgUnitPrice` | ○ | 整数（3桁区切り） |
| 更新日時 | `updatedAt` | ○ | YYYY-MM-DD HH:mm |
| 操作 | - | - | 閲覧・編集・削除ボタン |

ソートはカラムヘッダークリックで切替（同じカラム再クリックで昇順/降順反転）。

### ページネーション仕様

| 項目 | 値 |
|---|---|
| デフォルト表示件数 | 20件/ページ |
| ページ番号 | 1始まり |
| 表示形式 | 「全N件中 X〜Y件を表示」 |

## 3.10 月次スケジュールタブ（`TabSchedule.vue`）

### 概要

活動種別（行）× 月（列）のマトリクス形式でスケジュールを管理します。

### 列構成（月）

| 区分 | 月 |
|---|---|
| 上期 | 3月・4月・5月・6月・7月・8月・9月 |
| 下期 | 10月・11月・12月・1月・2月 |

下期の列は背景色で視覚的に区別します。

### デフォルト活動種別（6行）

1. 顧客イベント
2. SI計画（作業時間）
3. 交代要員（BP）戦略
4. BP単価UP戦略
5. FY27戦略
6. ゴール戦略（継続）

### 編集機能（編集モードのみ）

| 機能 | 説明 |
|---|---|
| セル入力 | 各セルにテキストエリアで自由入力 |
| 活動種別リネーム | 種別名をクリックして直接編集（blur時確定） |
| 行追加 | 「+ 行を追加」ボタンで末尾に追加 |
| 行削除 | 各行左端の「✕」ボタン（確認ダイアログあり） |

### 閲覧モード

- セルはテキストエリアではなく `div` で表示（内容量に応じて高さ自動伸縮）
- 行の追加・削除・リネームは不可
- 「活動種別」列と月ヘッダー行はスクロール時に固定表示（CSS sticky）

## 3.11 ユーザー管理画面（`/users`）

### 画面構成

1. 共通ヘッダー（「← 一覧に戻る」リンク）
2. ユーザー一覧テーブル
3. 新規ユーザー追加フォーム

### ユーザー一覧テーブルの表示項目

| カラム | 説明 |
|---|---|
| 氏名 | ユーザー氏名 |
| メールアドレス | ログインID |
| 権限 | 管理者 / 一般 |
| ステータス | 有効 / 無効 |
| ロック状態 | ロック中の場合は解除ボタン表示 |
| 操作 | 編集・パスワード変更・削除ボタン |

### 新規ユーザー追加フォーム

| フィールド名 | 型 | 必須 | バリデーション |
|---|---|---|---|
| 氏名 | テキスト | **○** | - |
| メールアドレス | email | **○** | @sas-com.com ドメインのみ |
| 初期パスワード | テキスト | **○** | 8文字以上・英字含む・数字含む |
| 権限 | セレクトボックス | **○** | `一般ユーザー`(user) / `管理者`(admin) |

### ユーザー編集フォーム

| フィールド名 | 型 | 必須 |
|---|---|---|
| 氏名 | テキスト | **○** |
| 権限 | セレクトボックス | **○** |
| ステータス | セレクトボックス | - |

### 制約事項

- 自分自身のアカウントは削除不可（削除ボタンが非活性）
- パスワード変更は管理者が強制的に変更する形式（本人確認なし）
- パスワードリセット時はロック状態も自動解除

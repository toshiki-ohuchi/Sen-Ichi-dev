# 4. データベース設計

## 4.1 ER図

```
┌──────────────────────────┐
│          records          │
│  PK id (serial)           │
│     no                    │
│     customer_name         │
│     ... (各種フィールド)    │
│     version               │
│     created_at / by       │
│     updated_at / by       │
└───────────┬──────────────┘
            │ 1
            │
            ├──────────────────────────────────────┐
            │ 多                                    │ 多
┌───────────▼────────────┐          ┌──────────────▼──────────────┐
│          todos          │          │      monthly_schedules       │
│  PK id (serial)         │          │  PK id (serial)              │
│  FK record_id → records │          │  FK record_id → records      │
│     agenda              │          │     month                    │
│     assignee            │          │     activity_type            │
│     start_date          │          │     content                  │
│     end_date            │          │                              │
│     exec_date           │          │  UNIQUE(record_id, month,    │
│     result              │          │         activity_type)       │
│     activity_check      │          └──────────────────────────────┘
│     sort_order          │
└─────────────────────────┘

┌──────────────────────────┐     ┌──────────────────────────────┐
│          users            │     │          sessions             │
│  PK id (serial)           │     │  PK id (varchar 100)          │
│     email (UNIQUE)        │     │     user_email               │
│     password_hash         │     │     user_name                │
│     name                  │     │     user_role                │
│     role                  │     │     expires_at               │
│     is_active             │     │     created_at               │
│     failed_login_count    │     └──────────────────────────────┘
│     locked_until          │
│     last_login_at         │
│     created_at / by       │
└──────────────────────────┘
```

**リレーション：**
- `records` → `todos`：1対多（`todos.record_id` が `records.id` を参照、CASCADE DELETE）
- `records` → `monthly_schedules`：1対多（同上、CASCADE DELETE）
- `users` と `sessions` は直接のFK関係なし（`sessions.user_email` で紐付け）

### 4.2.3 monthly_schedules テーブル

月次活動スケジュールのセルデータ。活動種別×月の組み合わせで1レコード。

| カラム名 | TSフィールド名 | 型 | NULL | デフォルト | 説明 |
|---|---|---|---|---|---|
| `id` | `id` | serial | NOT NULL | auto increment | **主キー** |
| `record_id` | `recordId` | integer | **NOT NULL** | - | **FK → records.id（CASCADE DELETE）** |
| `month` | `month` | varchar(10) | **NOT NULL** | - | 月（例：「3月」「10月」） |
| `activity_type` | `activityType` | varchar(100) | **NOT NULL** | - | 活動種別名 |
| `content` | `content` | text | NULL | - | セル内容テキスト |

**ユニーク制約：** `(record_id, month, activity_type)` の複合ユニークインデックス `monthly_schedules_unique`

## 4.3 インデックス・制約一覧

| テーブル | 制約種別 | 対象カラム | 制約名 |
|---|---|---|---|
| `users` | UNIQUE | `email` | （Drizzle自動付与） |
| `monthly_schedules` | UNIQUE | `(record_id, month, activity_type)` | `monthly_schedules_unique` |
| `todos` | FOREIGN KEY | `record_id → records.id` | CASCADE DELETE |
| `monthly_schedules` | FOREIGN KEY | `record_id → records.id` | CASCADE DELETE |

## 4.4 DB接続仕様

```typescript
// api/db/index.ts
import { neon } from '@neondatabase/serverless'
import { drizzle } from 'drizzle-orm/neon-http'

const sql = neon(process.env.DATABASE_URL!)
export const db = drizzle(sql, { schema })
```

- **接続方式：** HTTP（WebSocketではなくHTTPプロトコルで接続）
- **接続先：** 環境変数 `DATABASE_URL` に設定されたNeon接続文字列
- **コネクションプール：** Neonのサーバーレス接続が自動管理（Serverless Functionの特性上、常駐接続なし）

# 6. 認証・セキュリティ設計

## 6.1 認証方式の概要

本システムはセッションCookieによる認証を採用します。

| 項目 | 採用方式 | 理由 |
|---|---|---|
| 認証方式 | セッションCookie | シンプルで堅牢。JWTと異なりサーバー側でセッション無効化が可能 |
| パスワード保存 | bcryptjs（saltRounds=10） | 不可逆なハッシュ化でDB漏洩時もパスワード保護 |
| セッション管理 | DBテーブル（sessions）で管理 | セッション無効化・一覧確認が可能 |

---

## 6.2 ログイン処理フロー

```
クライアント          Hono API               PostgreSQL
    │                    │                        │
    │── POST /auth/login ▶│                        │
    │   {email, password} │                        │
    │                    ├── SELECT user by email ▶│
    │                    │◀─────────────────────── │
    │                    │                        │
    │                    ├─ [タイミング攻撃対策]      │
    │                    │  ユーザー不在でも         │
    │                    │  bcrypt.compareを実行   │
    │                    │                        │
    │                    ├─ [ロックチェック]          │
    │                    │  lockedUntil > 現在時刻  │
    │                    │  → 423返却             │
    │                    │                        │
    │                    ├─ [パスワード照合]          │
    │                    │  bcrypt.compare()       │
    │                    │  不一致 → failedCount++ │
    │                    │  5回以上 → ロック設定+423 │
    │                    │  一致 → 次のステップへ   │
    │                    │                        │
    │                    ├─ [アカウント有効チェック]   │
    │                    │  isActive=false → 403  │
    │                    │                        │
    │                    ├─ セッション生成            │
    │                    │  randomBytes(32).hex()  │
    │                    ├── INSERT sessions ──────▶│
    │                    │                        │
    │◀─ 200 + Cookie ────│                        │
    │   session_id=xxx    │                        │
```

## 6.3 セッション仕様

| 項目 | 値 |
|---|---|
| Cookie名 | `session_id` |
| セッションID生成方式 | `randomBytes(32).toString('hex')`（64文字の16進数） |
| 有効期限 | **7日間**（ログイン時刻から7日後） |
| httpOnly | **true**（JavaScriptからアクセス不可） |
| secure | **true**（本番環境のみ。HTTPS通信時のみ送信） |
| sameSite | **Lax**（CSRF対策） |
| path | `/` |

## 6.4 認証ミドルウェア（authMiddleware）

全保護APIに適用される認証チェック処理。

```
リクエスト受信
    │
    ├─ Cookie 'session_id' の取得
    │  → なければ 401 Unauthorized
    │
    ├─ sessionsテーブルからセッション検索
    │  → 存在しなければ 401 Unauthorized
    │
    ├─ セッション有効期限チェック
    │  expiresAt < 現在時刻 → 401 'Session expired'
    │
    ├─ コンテキストにユーザー情報をセット
    │  c.set('userEmail', session.userEmail)
    │  c.set('userName', session.userName)
    │  c.set('userRole', session.userRole)
    │
    └─ next() → 実際のAPIハンドラへ
```

## 6.5 アカウントロック仕様

ブルートフォース攻撃（パスワードの総当たり攻撃）への対策として実装。

| 項目 | 値 |
|---|---|
| 最大ログイン失敗回数（MAX_FAILED） | **5回** |
| ロック時間（LOCK_MINUTES） | **15分** |
| ロック解除条件 | 時間経過による自動解除 **または** 管理者が `PUT /api/users/:id/unlock` を実行 |
| パスワードリセット時の挙動 | `failedLoginCount=0`・`lockedUntil=null` に自動リセット |

### ロック状態の遷移

```
ログイン成功          → failedLoginCount=0, lockedUntil=null にリセット
ログイン失敗（1〜4回）→ failedLoginCount インクリメント
ログイン失敗（5回目）→ lockedUntil = 現在 + 15分 をセット → 423返却
ロック中のログイン試行→ 423返却（failedLoginCountはインクリメントしない）
15分経過            → lockedUntil < 現在時刻 → 自動的にロック解除
```

## 6.6 パスワードポリシー

| 項目 | ルール |
|---|---|
| 最小文字数 | 8文字以上 |
| 英字 | 含む必須（`/[a-zA-Z]/` にマッチ） |
| 数字 | 含む必須（`/[0-9]/` にマッチ） |
| 適用箇所 | ユーザー新規登録・パスワード変更 |

ハッシュ化：`bcrypt.hash(password, 10)`（saltRounds=10）

## 6.7 ユーザー登録制限

メールアドレスのドメインを `@sas-com.com` のみに制限することで、外部者のアカウント作成を防止。

```typescript
const ALLOWED_DOMAIN = 'sas-com.com'
if (!email.endsWith('@' + ALLOWED_DOMAIN)) {
  return error(400, '@sas-com.com のメールアドレスのみ登録できます。')
}
```

メールアドレスはDB保存時に小文字に正規化（`email.toLowerCase()`）。

## 6.9 タイミング攻撃対策

ユーザーが存在しない場合と、存在するがパスワードが一致しない場合で、レスポンス時間に差が生じないよう対策。

```typescript
// ユーザー不在時もダミーハッシュとbcrypt.compareを実行して応答時間を均一化
const DUMMY_HASH = '$2b$10$dummy.hash.for.timing.attack.prevention'
const isMatch = user
  ? await bcrypt.compare(password, user.passwordHash)
  : await bcrypt.compare(password, DUMMY_HASH)
```

## 6.10 その他セキュリティ考慮事項

| 項目 | 対策 |
|---|---|
| XSS | `httpOnly` CookieでセッションIDをJSから隔離 |
| CSRF | `sameSite: Lax` で異なるオリジンからのリクエストにCookieが送信されない |
| SQLインジェクション | Drizzle ORMのパラメータバインドを使用（生SQL文字列連結なし） |
| 平文パスワード保存 | bcryptハッシュのみDB保存（元パスワードは保存しない） |
| CORS | `BASE_URL` 環境変数で許可オリジンを限定 |
| セッション固定 | ログイン時に新規セッションIDを生成（randomBytes） |

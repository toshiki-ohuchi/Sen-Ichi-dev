# 8. 開発・運用

## 8.1 ローカル開発環境の構築手順

### 前提条件

| ツール | バージョン |
|---|---|
| Node.js | 18以上 |
| npm | 9以上 |
| Git | 最新版 |
| Neonアカウント | DB接続文字列が必要 |

### セットアップ手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/toshiki-ohuchi/Sen-Ichi-dev.git
cd Sen-Ichi-dev

# 2. 依存パッケージのインストール
npm install

# 3. 環境変数ファイルの作成
cp .env.example .env   # .env.example がある場合
# または直接 .env を作成
```

### 環境変数（`.env`）

```
DATABASE_URL=postgresql://user:password@host/dbname?sslmode=require
BASE_URL=http://localhost:5173
```

- `DATABASE_URL`：Neonのコンソールから取得した接続文字列
- `BASE_URL`：フロントエンドのURL（CORS許可用）

### 起動方法

```bash
# フロントエンド開発サーバー（http://localhost:5173）
npm run dev

# APIサーバー（Vercel CLI使用時）
npx vercel dev
```

## 8.2 DBマイグレーション手順（Drizzle Kit）

スキーマ変更（カラム追加・型変更等）を行う場合の手順です。

### 手順

```bash
# 1. api/db/schema.ts を編集してスキーマを変更

# 2. マイグレーションを実行（直接DBに適用）
npm run db:push

# または差分SQLを確認してから適用
npx drizzle-kit generate   # マイグレーションファイル生成
npx drizzle-kit migrate    # 適用
```

### 注意事項

- `db:push` は開発環境での確認に使用。本番環境では慎重に実行する
- カラム削除・型変更は既存データへの影響を事前に確認する
- `DATABASE_URL` が正しい環境（本番/開発）を指していることを確認する

## 8.3 Vercelデプロイ手順

### 自動デプロイ（推奨）

`main` ブランチへのプッシュで自動デプロイされます。

```bash
git push origin main   # → Vercelが自動でビルド・デプロイ
```

### デプロイの仕組み

1. Vercelが `main` ブランチの更新を検知
2. `npm run build`（`vite build`）を実行してフロントエンドをビルド
3. `api/server.ts` がServerless Functionとしてデプロイ
4. `vercel.json` のルーティング設定が適用される

### Vercel管理画面での確認事項

| 確認項目 | 場所 |
|---|---|
| ビルドログ | Vercel Dashboard > Deployments |
| 環境変数 | Vercel Dashboard > Settings > Environment Variables |
| 関数ログ | Vercel Dashboard > Functions |

## 8.7 障害対応

### ログの確認方法

| ログ種別 | 確認方法 |
|---|---|
| APIエラーログ | Vercel Dashboard > Functions > ログタブ |
| フロントエンドエラー | ブラウザの開発者ツール（Console・Network） |
| DBエラー | Vercel Functionログ（Drizzleのエラーメッセージ） |

### よくある障害と対応

| 症状 | 考えられる原因 | 対応 |
|---|---|---|
| ログインができない（500エラー） | `DATABASE_URL` の設定誤り | Vercel環境変数を確認・再設定 |
| ログインできない（423エラー） | アカウントロック中 | 管理者が `/users` からロック解除 |
| 更新で「競合エラー」が出る | 同時編集による楽観的ロック発動 | 画面をリロードして最新データで再編集 |
| APIが404を返す | Vercelのルーティング設定の問題 | `vercel.json` の設定を確認 |
| DBマイグレーション後に画面が崩れる | スキーマとフロントエンド型の不整合 | `src/types/index.ts` とスキーマを同期確認 |

## 8.8 データバックアップ

Neonのデータベースはバックアップ機能をNeon側で提供しています。

| 項目 | 内容 |
|---|---|
| バックアップ方式 | Neonのブランチ機能・自動バックアップ |
